GIT学习笔记

一些概念 工作区: 包含.git文件夹的目录就是工作区，也就是你的项目文件夹 暂存区: .git文件夹中的index文件是暂存区,也叫做stage，是临时保存修改文件的地方 版本库: .git文件夹是版本库，版本库中存储了许多配置信息，日志信息，文件版本等 流程: 工作区发生改变后使用命令[ git add . ] 将修改临时保存到暂存区, 使用命令 [ git commit -m “提交信息”] 提交到版本库 文件状态 untracked 未跟踪 [未被纳入版本控制] tracked [已被纳入版本控制] Unmodified 未修改状态 Modified 已修改状态 Staged 已暂存状态 常用命令 全局设置 # 全局设置 # 设置用户名 git config --global user.name "用户名" # 设置用户邮箱 git config --global user.email "邮箱地址" 仓库操作 # 新建一个目录，将其初始化为Git代码库 git init [project-name] # 查看文件状态 git status # 将文件的修改添加到暂存区 git add [file1][file2]... # 删除工作区文件, 并将这次操作放入暂存区 git rm [file1][file2]....
