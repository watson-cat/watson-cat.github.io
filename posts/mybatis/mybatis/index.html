简介: 一个用于数据持久层的框架

基本配置#

//mybatis配置文件
String resource = "mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
try (SqlSession session = sqlSessionFactory.openSession()) {
    //此处是直接调用方法进行操作，建议使用代理开发
    List<EoeAccount> datas = session.selectList("test.selectAll");
    datas.forEach(System.out::println);
    session.close();
}

Mybatis配置文件(mybatis-config.xml)

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "https://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/eoezora?useSSL=false&amp;allowPublicKeyRetrieval=true"/>
                <property name="username" value="root"/>
                <property name="password" value="root"/>
            </dataSource>
        </environment>
    </environments>
    <mappers>
        <mapper resource="mappers/TestDemoMapper.xml"/>
    </mappers>
</configuration>

SQL语句

映射文件(TestDemoMapper.xml)

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!-- 代理开发中 namespace的值应该是 Mapper接口的全限定名 -->
<!-- 例如 com.eoezora.mapper.TestDemoMapper -->
<mapper namespace="test">
    <!-- resultType 返回值 -->
    <select id="selectAll" resultType="com.eoezora.pojo.EoeAccount">
        SELECT * FROM account;
    </select>
</mapper>

Mapper代理开发#

定义与SQL映射文件同名的Mapper接口,并且将Mapper接口与SQL映射文件放置在同名文件下

设置SQL映射文件的namespace属性与Mapper接口全限定名

在Mapper接口中定义方法,方法名是SQL映射文件中的sql语句ID，并保持参数类型和返回值类型一致

try (SqlSession session = sqlSessionFactory.openSession()) {
    //获取Mapper接口
    TestDemoMapper testDemoMapper = session.getMapper(TestDemoMapper.class);
    //使用接口调用方法
    testDemoMapper.selectAll().forEach(System.out::println);
}

设置别名#

在Nybatis的sql的映射文件中,会使用paramterType、resultType 来设置 sql 语句的输入输出参数，一般参数都是基本的数据类型或封装类型，但都需要声明该类型的全路径，java.lang.String，或者 cn.com.mybatis.pojo.User, 这时就可以通过 typeAliases设置别名,简化复杂度，提高阅读性, 例子如下

<select id="selectUserByName" parameterType="java.lang.String" resultType="com.mybatis.pojo.User">
    SELECT * FROM USER WHERE username LIKE '%${value}%'
</select>

使用别名简化后

<typeAliases>
    <typeAlias alias="user" type="com.mybatis.pojo.User"/>
    <typeAlias alias="string" type="java.lang.String"/>
</typeAliases>
<!-- paramterType、resultType直接使用别名 -->
<select id="selectUserByName" parameterType="string" resultType="user">
    SELECT * FROM USER WHERE username LIKE '%${value}%'
</select>

其他

批量添加别名,以下设置,会为该包下的所有类自动添加别名,为对应类名的首字母小写 User -> user

<typeAliases>
    <package name="com.mybatis.pojo"/>
</typeAliases
