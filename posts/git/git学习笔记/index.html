一些概念#

工作区: 包含.git文件夹的目录就是工作区，也就是你的项目文件夹

暂存区: .git文件夹中的index文件是暂存区,也叫做stage，是临时保存修改文件的地方

版本库: .git文件夹是版本库，版本库中存储了许多配置信息，日志信息，文件版本等

流程: 工作区发生改变后使用命令[ git add . ] 将修改临时保存到暂存区, 使用命令 [ git commit -m “提交信息”] 提交到版本库

文件状态#

  • untracked 未跟踪 [未被纳入版本控制]
  • tracked [已被纳入版本控制]
    • Unmodified 未修改状态
    • Modified 已修改状态
    • Staged 已暂存状态

常用命令#

全局设置#

# 全局设置
# 设置用户名
git config --global user.name "用户名"
# 设置用户邮箱
git config --global user.email "邮箱地址"

仓库操作#

# 新建一个目录，将其初始化为Git代码库
git init [project-name]

# 查看文件状态
git status

# 将文件的修改添加到暂存区
git add [file1][file2]...

# 删除工作区文件, 并将这次操作放入暂存区
git rm [file1][file2]...

# 停止追踪指定文件，但该文件会保留在工作区
git rm --cached [file]

# 文件改名，这次操作放入暂存区
git mv [file-original] [file-renamed]

# 将暂存区的文件取消暂存或者切换到指定版本
git reset [file]

# 将暂存区的文件修改提交到版本区
git commit -m "提交信息" [file]
git commit [file1][file2]... -m "提交信息" 

# 查看日志
git log

# 查看远程仓库
git remote

# 添加远程仓库
git remote add [shotName] [url]

# 从远程仓库克隆
git clone [url]

# 从远程仓库拉取
git pull

# 推送到远程仓库
git push -u [shotName] [branchName]

分支操作#

# 查看分支
# 不加参数 列出所有本地分支
# -r 列出所有远程分支
# -a 列出本地跟远程的所有分支
git branch [-r/-a]

# 创建分支
git branch [name]

# 新建分支并切换
git checkout -b [name]

# 推送到远程仓库分支
git push [shortName] [name]

# 合并分支
git merge [name]
